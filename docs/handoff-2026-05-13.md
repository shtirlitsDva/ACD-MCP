<misc-gotchas>
- DevReload's `%APPDATA%\DevReload\plugins.json` for `Acd.Mcp` has `loadOnStartup: false`. The autonomous-bootstrap doc instructs the agent to flip it to `true` and restore on teardown. This session's run left it at `false` (restored).
- The `Application.Idle` auto-start hook in `McpPlugin.Initialize` was a TEMPORARY v18 verification aid. Already reverted at commit `22f8b3b`. **Do not re-add without `#if DEBUG`** â€” auto-starting the listener in production hands every Civil 3D session a pipe whether the user wants it or not.
- Civil 3D licensing is one-instance-per-Windows-session. If the user has Civil 3D open visibly, a `/Automation` launch silently blocks on a license popup the agent can't see. Always check `Get-Process acad` first and match-by-PID to discriminate user processes from agent-launched ones.
- The G3/G4 path benefits from `Test-Path \\.\pipe\NAME` being unreliable for named-pipe readiness (returns false for valid-but-unconnected pipes). Use `[System.IO.Directory]::GetFiles("\\\\.\\pipe\\")` instead. Documented in the autonomous-bootstrap section.
</misc-gotchas>
